设计模式：简单工厂模式Simple Factory Pattern

1. 简单工厂模式也叫静态工厂方法模式，是工厂方法模式的一种特例

2. 简单工厂方法模式解决什么问题？
    客户端程序不需要关心对象的创建细节，需要哪个对象时，
    只需要向工厂索要即可，初步实现了责任的分离。客户端只负责“消费”，工厂负责“生产”。生产和消费分离。
3. 简单工厂模式中的角色：
    抽象产品角色
    具体产品角色
    工厂类角色
4. 缺点
    ● 缺点1：工厂类集中了所有产品的创造逻辑，形成一个无所不知的全能类，有人把它叫做上帝类。
    显然工厂类非常关键，不能出问题，一旦出问题，整个系统瘫痪。
    ● 缺点2：不符合OCP开闭原则，在进行系统扩展时，需要修改工厂类。
